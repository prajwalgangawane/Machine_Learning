Data Normalization and Standarization We can either :

  1. Remove outlier using Inter-quartile range or z-score, however removing outlier will cause loss of data.
  2. Capping outliers to mix and max value.
  3. Replace outliers with mean/median and treat them as missing values.

Since most of the outlier values seem to lie on edges so we will choose to replace all high values where (mean + 3 std) and low values where (mean - 3 std) with capped values. Tuning the calculation we find that capping values to (mean + 2 std) and (mean - 2 std.) reduce the outliers

We will leave out stream_velocity_mps from scaling as it has only 4 unique values.

#replacing columns with capped upped and lower bounds
for column_name in df_self_noise.columns:
  replace_value = df_self_noise[column_name].mean()
  lower_limit = df_self_noise[column_name].mean() - 2 * df_self_noise[column_name].std()
  upper_limit = df_self_noise[column_name].mean() + 2 * df_self_noise[column_name].std()
  print(column_name,lower_limit, upper_limit)
  if column_name not in ['stream_velocity_mps']:
    df_self_noise[column_name] = np.where(df_self_noise[column_name] > upper_limit,upper_limit,
                               np.where(df_self_noise[column_name] < lower_limit,lower_limit,
                                        df_self_noise[column_name]))

Standardization, Normalization

To ensure that the gradient descent moves smoothly towards the minima and that the steps for gradient descent are updated at the same rate for all the features, we scale the data before feeding it to the model.

  1. Since data contains outliers we will use Normalization using MinMaxScaler
  2. We will also use Standarization using StandardScaler to study the affect on Model performance.
